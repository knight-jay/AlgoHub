# AlgoHub 项目功能说明

## 一、项目概述

**项目名称**：AlgoHub——算法学习一站式服务平台

**项目目标**：为算法学习者和竞赛选手提供集算法知识库、学习资源推荐、问答社区于一体的综合服务平台，支持学生与管理员的角色权限管理，满足学习、交流与资源获取的全流程需求。

**技术架构**：
- 前端：独立前端应用（目录 `src/frontend/`）
- 后端：RESTful API（目录 `src/backend/`）
- 数据库：SQL脚本（目录 `sql/`）
- 文档：计划书、设计文档、测试材料、会议纪要、说明书等（`docs/` 下分模块）
- UI资源：原型图、界面素材（`ui/`）

**网站整体布局**：
1. 未登录状态：仅显示登录页面。
2. 登录后进入主界面，主界面底部或顶部导航包含三大板块：
   - **问答社区**（主界面默认首页）
   - **学习资源推荐**
   - **个人信息**
3. 右上角全局搜索按钮：**算法知识库搜索**（按算法名称检索）。

## 二、用户体系

### 2.1 用户注册

- **注册方式**：手机号
- **可选角色**：学生 / 管理员
- **校验规则**：
  - 手机号格式合法性
  - 手机号唯一性
  - 用户名自动生成或允许填写，后续可修改昵称
- **流程**：填写信息 → 系统校验 → 创建账号 → 自动跳转至登录页面

### 2.2 用户登录

- **登录凭证**：用户名 + 密码
- **成功跳转**：平台首页（问答社区）
- **失败提示**：用户名或密码错误

### 2.4 个人信息管理

| 可修改字段                         | 不可修改字段 |
| ---------------------------------- | ------------ |
| 手机号、个人简介、昵称、帖子、收藏夹 | 用户名       |

**修改密码流程**：
1. 输入原密码
2. 输入新密码
3. 确认新密码
4. 原密码错误时提示：“原密码错误”

**效果**：信息修改成功后，前端展示的用户信息实时更新。

### 2.5 角色权限控制

#### 学生权限
- 浏览算法知识库
- 浏览学习资源
- 发布/删除自己的帖子
- 评论/点赞/关注/收藏
- 举报内容
- 修改个人信息及密码

#### 管理员权限（包含学生所有权限，并额外拥有）
- **用户管理**：禁用/启用用户账号，修改用户角色
- **内容管理**：
  - 管理算法知识库（增删改算法条目）
  - 管理学习资源（增删改推荐网站）
- **社区管理**：
  - 审核问答内容（删除违规帖子/评论）
  - 处理举报（查看举报列表，裁定处理）

## 三、算法知识库

### 3.1 访问入口
- 主界面右上角 **搜索按钮** → 输入算法名称（如“快速排序”） → 展示结果

### 3.2 数据来源
- 预置数据文件：`init_algorithms.sql`（包含算法名称、简介、用法、C++模板代码）

### 3.3 展示内容
- 算法简介
- 算法用法说明
- C++ 模板代码（代码块支持一键复制）

### 3.4 扩展功能（可选，基于树形分类目录）
- 分类筛选：如动态规划、图论、数据结构等
- 多维度检索：按标签、难度等

## 五、问答社区（主板块）

### 5.1 帖子列表页
- 默认进入社区主界面，展示所有帖子
- 排序方式：
  - **按时间顺序**（最新发布在前）
  - **按热度排序**（帖子点赞总数倒序）
- 每个帖子条目显示：标题、作者、发布时间、点赞数、评论数

### 5.2 帖子详情页
- 帖子内容（文字、代码等）
- 操作：
  - 点赞 / 取消点赞
  - 关注帖子（收到新评论通知）
  - 关注其他用户 → 可在个人页查看被关注者的最新帖子（预留）
  - 收藏帖子 → 存入个人收藏夹
  - 举报帖子（触发管理员审核）
  - 评论：用户可发表评论，其他用户可回复评论

### 5.3 个人帖子管理
- 用户可在“个人信息”页面管理自己的帖子：
  - 编辑
  - 删除

### 5.4 管理员审核与处置
- 查看所有举报列表
- 对违规帖子或评论进行：
  - 删除（同时扣除发布者信用或记录）
  - 驳回举报并标记为无效
- 直接删除任何帖子（管理面板）

### 5.5 帖子搜索
- 在社区顶部搜索框输入关键词
- 按标题和内容进行关键词匹配
- 搜索结果分页展示，按时间倒序排列

### 5.6 发布帖子
- 所有登录用户均可发布帖子
- 必须填写标题
- 内容可为文字或代码（支持代码块）
- 发布后自动跳转至帖子详情页
