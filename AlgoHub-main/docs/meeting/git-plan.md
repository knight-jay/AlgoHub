# Git 阶段计划

## 1. 分支策略

- `main`：稳定最终版本
- `dev`：日常开发集成分支
- `feature/姓名-模块`：功能开发分支，例如 `feature/zhengxinjie-knowledge`

## 2. 提交规范

- 每次提交使用清晰说明，例如：
  - `[用户] 完成注册接口`
  - `[知识库] 实现检索功能`
  - `[文档] 补充测试记录`
- 重要阶段完成后打 tag：`v1-plan`、`v2-design`、`v3-final`
