### 2.2 各包职责详解

#### `common/` — 通用响应封装

所有 Controller 通过这两个类返回 JSON，保证前后端数据格式统一。

| 文件 | 作者 | 作用 |
|---|---|---|
| `Result.java` | junkfood24 | 统一 API 响应体。三个字段：`code`（200成功/401未登录/403无权限/500错误）、`msg`（提示信息）、`data`（泛型数据）。静态工厂方法 `Result.success(data)` / `Result.error(code, msg)` |
| `PageResult.java` | **KnightJay** `52f6dc3` | 分页查询统一响应体。四个字段：`list`（当前页数据）、`total`（总条数）、`page`（当前页码）、`pageSize`（每页条数） |

#### `config/` — 配置类

| 文件 | 作者 | 作用 |
|---|---|---|
| `WebConfig.java` | junkfood24 初建，**KnightJay** 多次修改 | 实现 `WebMvcConfigurer`，注册 JwtInterceptor。核心逻辑：拦截所有 `/api/**` → 放行白名单路径 → 其余校验 JWT |

**白名单策略**（我参与演进）：
```java
// junkfood24 初建：放行 login/register
// KnightJay 07776c2：修复 — 移除对 /api/admin/** 的错误排除
// KnightJay 6229821：扩展 — 增加 /api/resources/** 公开浏览
// KnightJay 31af5a4：扩展 — 增加 /api/users/* 用户公开信息
excludePathPatterns(
    "/api/user/login", "/api/user/register", "/api/user/forgot-password",
    "/api/algorithm/**",      // 算法知识库公开
    "/api/resources/**",       // 学习资源公开
    "/api/posts/**",           // 社区公开
    "/api/comments/**",
    "/api/users/*/follow", "/api/users/*", "/api/users/*/posts"
);
```

#### `interceptor/` — 拦截器

| 文件 | 作者 | 作用 |
|---|---|---|
| `JwtInterceptor.java` | junkfood24 初建，**KnightJay** `653f803` 重写 | `preHandle()`：从 `Authorization: Bearer <token>` 提取 Token → 校验过期 → 解析 userId/username/role → 注入 `request.setAttribute()`。校验失败返回 401 JSON |

#### `util/` — 工具类

| 文件 | 作者 | 作用 |
|---|---|---|
| `JwtUtil.java` | junkfood24 初建，junkfood24 `ed3279c` 扩展 | JWT 工具：`generateToken(userId, username, role)` 默认24h过期、"记住我"7天过期（`REMEMBER_ME_EXPIRATION`）、`extractClaims(token)` 解析载荷、`isTokenExpired(token)` 校验。密钥从 `application.yml` 读，HS256 签名 |

#### `exception/` — 异常处理

| 文件 | 作者 | 作用 |
|---|---|---|
| `GlobalExceptionHandler.java` | junkfood24 初建，**KnightJay** `653f803` 重写 | `@RestControllerAdvice`：`ConnectException`→网络错误、`ExpiredJwtException`→401登录过期、`SignatureException/MalformedJwtException/IllegalArgumentException`→401状态异常、兜底 `Exception`→500服务器错误。**这是我修的 Bug #1**——之前所有异常都返回"注册失败" |

## 六、技术要点总结

1. **三层架构 + 接口分离**：Controller → Service(Interface→Impl) → Repository（Spring Data JPA），每层职责明确，接口与实现分离
2. **统一响应格式**：`Result<T>` + `PageResult<T>` 保证前后端接口契约一致，前端只需判断 `code` 字段
3. **三级角色权限**：STUDENT（学生）< ADMIN（管理员）< MASTER（群主），层级保护——不可操作同级或更高级用户
4. **JWT 无状态认证**：拦截器 + 白名单模式，公开接口放行，认证接口从 `request.getAttribute` 读取用户信息
5. **分页查询**：全部列表接口支持 page/pageSize，`PageRequest` + `Page<T>` 自动统计总数
6. **关键词搜索**：`@Query` + SQL `LIKE` 联合搜索 title、description、tags 字段
7. **手动 DDL 管理**：`ddl-auto: none`，表结构全部由 SQL 脚本管理，版本可控
8. **密码安全**：MD5 摘要存储，查询接口统一 `setPassword(null)` 脱敏
