二、总体需求概述
2.1 项目背景与初期定位
在AI替代基础编程任务的背景下，平台聚焦在校计算机专业大学生的算法学习需求。用户体系是平台的基础底座，所有后续模块（知识库、问答、资源）均依赖用户身份认证与权限控制。初期优先保证用户体系的稳定性、安全性与易用性，同时磨合团队协作流程。
2.2 初期非功能需求
需求类型	具体指标
性能要求	登录/注册接口响应时间≤200ms；页面首屏加载时间≤1.5s；支持50人同时在线
安全要求	用户密码BCrypt加密存储；JWT令牌有效期24小时；敏感操作（修改密码）需二次验证；输入XSS过滤与SQL防注入
兼容性要求	支持Chrome 90+、Edge 90+浏览器；1920×1080分辨率下界面无错乱
可用性要求	核心流程（注册→登录→修改信息→退出）成功率100%；无阻塞性Bug
三、用户体系详细功能需求
3.1 用户注册
功能描述
支持学生通过邮箱/手机号完成账号注册，系统校验信息合法性并创建用户账号，默认分配"学生"角色。
业务规则
1.注册字段：用户名（唯一，3-20位字母/数字/下划线）、密码（6-20位，含字母+数字）、确认密码、邮箱（唯一，用于找回密码）、手机号（选填）
2.校验规则：
--用户名/邮箱已存在时，提示"该用户名/邮箱已被注册"
--密码与确认密码不一致时，提示"两次输入密码不一致"
==邮箱格式错误时，提示"请输入有效的邮箱地址"
3.注册成功后，自动跳转至登录页面；注册失败保留已输入信息（密码除外）
异常处理
网络异常：提示"网络错误，请稍后重试"
服务器错误：提示"注册失败，请联系管理员"
3.2 用户登录
功能描述
支持用户通过"用户名+密码"登录，登录成功后生成JWT令牌，跳转至平台首页。
业务规则
1.登录字段：用户名/邮箱、密码、"记住我"（可选，勾选后令牌有效期延长至7天）
2.校验规则：
--用户名/邮箱不存在时，提示"账号不存在"
--密码错误时，提示"密码错误，还可尝试X次"（连续5次错误锁定账号15分钟）
--账号被锁定时，提示"账号已被锁定，请15分钟后再试"
3.登录成功后，前端存储JWT令牌与用户基本信息（用户名、角色、头像）
4.未登录用户访问需认证页面时，自动跳转至登录页面
异常处理
令牌过期：自动跳转至登录页面，提示"登录已过期，请重新登录"
无效令牌：提示"登录状态异常，请重新登录"
3.3 找回密码
功能描述
支持用户通过注册邮箱接收验证码，重置账号密码。
业务规则
1.流程：输入用户名/邮箱→发送验证码→输入验证码→设置新密码→完成重置
2.验证码规则：6位数字，有效期5分钟，同一邮箱1分钟内只能发送1次
3.新密码需符合注册时的密码强度要求
4.重置成功后，自动跳转至登录页面，提示"密码重置成功，请重新登录"
异常处理
验证码错误：提示"验证码错误，请重新输入"
验证码过期：提示"验证码已过期，请重新获取"
邮箱未注册：提示"该邮箱未绑定任何账号"
3.4 个人信息管理
功能描述
登录用户可查看和修改个人基本信息，修改登录密码。
业务规则
1.可修改字段：昵称、头像（上传本地图片，限制2MB以内，支持JPG/PNG格式）、手机号、个人简介
2.不可修改字段：用户名、注册邮箱、用户ID
3.修改密码：需输入原密码→输入新密码→确认新密码，原密码错误时提示"原密码错误"
4.信息修改成功后，实时更新前端展示的用户信息
3.5 角色权限控制
功能描述
系统预设"学生"和"管理员"两种角色，不同角色拥有不同的操作权限。
权限划分
功能	学生角色	管理员角色
注册/登录/找回密码	✅	✅
查看/修改个人信息	✅	✅
访问算法知识库/资源推荐/问答社区	✅	✅
发布问题/回答/评论	✅	✅
管理用户账号（禁用/启用、修改角色）	❌	✅
管理算法知识库/学习资源	❌	✅
审核问答内容、处理举报	❌	✅
业务规则
1.新注册用户默认分配"学生"角色
2.管理员角色仅能由超级管理员（初始账号）在后台分配
3.前端根据用户角色隐藏无权限的功能入口；后端接口进行权限校验，越权操作返回403状态码
四、技术架构与开发规范
4.1 初期技术架构（前后端分离）
plain text

•工具链：Git+Gitee、Apifox、IDEA、VS Code、Navicat
4.2 Git协作规范
1.分支策略
--主分支`main`：仅存放稳定可运行版本，禁止直接提交
--开发分支`dev`：日常集成分支，所有功能分支从`dev`拉出
--功能分支：`feature/姓名-模块名`，如`feature/changyujie-user-auth`
2.提交规范：`[模块] 具体功能描述`，例如`[用户] 实现注册接口密码加密`
3.合并流程
--每日18:00前推送本地分支到远程，发起Pull Request到`dev`
--由架构师（常宇杰）审核代码后合并
--每日开工前执行`git pull origin dev`同步最新代码
4.冲突处理：遇到冲突先在本地解决，无法解决及时在群内反馈，禁止强制提交
4.3 接口规范（RESTful风格）
1.统一返回格式{
  "code": 200,
  "message": "操作成功",
  "data": {}
}状态码说明：200成功，400参数错误，401未登录，403无权限，500服务器错误
2.接口地址规范：`/api/v1/模块名/功能名`，例如`/api/v1/user/register`
3.请求方法：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）
4.所有接口必须提前在Apifox中定义，包含请求参数、响应参数、状态码说明，前后端确认后再开发
4.4 代码命名规范
--后端：类名大驼峰（UserController）、方法名小驼峰（registerUser）、变量名小驼峰、常量全大写（USER_STATUS_NORMAL）
--前端：组件名大驼峰（LoginForm.vue）、变量名小驼峰、路由路径短横线分隔（/user/profile）
--数据库：表名小写+下划线（sys_user）、字段名小写+下划线（user_name）

user_name	VARCHAR(20)	NOT NULL, UNIQUE	用户名

phone	VARCHAR(11)	NULL	手机号
nickname	VARCHAR(30)	NULL	昵称
avatar	VARCHAR(255)	NULL	头像URL

login_fail_count	INT	NOT NULL, DEFAULT 0	连续登录失败次数
lock_time	DATETIME	NULL	账号锁定时间
create_time	DATETIME	NOT NULL, DEFAULT CURRENT_TIMESTAMP	创建时间
update_time	DATETIME	NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP	更新时间
2. 角色表（sys_role）
字段名	数据类型	约束	说明
role_id	BIGINT	PRIMARY KEY, AUTO_INCREMENT	角色ID
role_name	VARCHAR(20)	NOT NULL, UNIQUE	角色名称：student/admin
role_desc	VARCHAR(50)	NULL	角色描述
create_time	DATETIME	NOT NULL, DEFAULT CURRENT_TIMESTAMP	创建时间
3. 用户角色关联表（sys_user_role）
字段名	数据类型	约束	说明
id	BIGINT	PRIMARY KEY, AUTO_INCREMENT	主键ID
user_id	BIGINT	NOT NULL, FOREIGN KEY REFERENCES sys_user(user_id)	用户ID
role_id	BIGINT	NOT NULL, FOREIGN KEY REFERENCES sys_role(role_id)	角色ID
UNIQUE KEY (user_id, role_id)	-	-	联合唯一索引
