## 1. 测试目标

- 验证管理员模块 3 个功能（获取用户列表、禁用/启用账号、修改用户角色）的正确性，确保管理员可正常进行用户管理操作。
- 验证权限隔离：学生身份访问管理员接口时返回 403，未登录访问时返回 401。
- 验证参数校验：非法 locked 值、非法 role 值、不存在的用户 ID 等异常输入被正确拦截。
- 验证安全约束：管理员不能禁用自身账号、不能降低自身权限（Bug 清单 #4、#5）。

## 3. 测试策略

### 3.1 测试类型
接口功能测试（黑盒），通过 Apifox 发送 HTTP 请求，校验 HTTP 状态码及响应 JSON 体。

### 3.2 用例设计方法
- **正向用例：** 管理员 Token + 合法参数，验证返回 `code: 200` 且操作生效（列表返回、禁用/启用、角色修改）。
- **异常用例——参数校验：** locked 传入非法值（999）、不传 locked、role 传入非法角色名（TEACHER）、用户 ID 不存在，验证返回 `code: 500` 并携带明确错误提示。
- **异常用例——鉴权校验：** 不带 Token，验证返回 `code: 401`。
- **异常用例——权限校验：** 学生 Token 访问管理员接口，验证返回 `code: 403`。
- **安全用例：** 管理员禁用自身账号、降低自身权限，验证被拦截（Bug 清单 #4、#5）。

### 3.3 测试流程
1. 预先准备管理员账号（admin1）和学生账号（test001），分别登录获取 Token。
2. 先执行正向用例确认功能可用，再执行越权/鉴权用例验证拦截。
3. 安全相关用例（自身操作拦截）参照 Bug 清单专项验证。
4. 不通过用例标注并同步追加至 Bug 清单，Bug 修复后回归验证。

## 6. 通过/失败标准

**通过标准：**
- 管理员正向操作返回 `code: 200`，业务逻辑生效（列表含 password=null、状态切换、角色修改）。
- 学生越权访问返回 `code: 403`，未登录访问返回 `code: 401`。
- 异常参数返回 `code: 500` 并携带可读错误提示。
- 管理员对自身进行禁用/降权操作被拦截。

**失败标准：**
- 学生可访问管理员接口（越权漏洞）。
- 非法参数未拦截导致数据异常。
- 管理员可禁用自身或降低自身权限（安全漏洞）。
- 用户列表泄露密码字段。

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 管理员自身操作未被拦截 | 管理员账号永久失效 | 专项验证自身禁用/降权场景（Bug #4、#5），确认拦截生效 |
| 用户列表返回敏感字段 | 密码泄露 | 验证返回列表中 password 字段为 null |
| Token 过期 | 鉴权用例误报 | 测试前重新登录获取有效 Token |
| 角色修改后未同步权限 | 用户权限状态不一致 | 修改角色后立即用该用户 Token 访问对应权限接口验证 |
