## 3. 测试策略

### 3.1 测试类型
接口功能测试（黑盒），通过 Apifox 发送 HTTP 请求，校验 HTTP 状态码及响应 JSON 体。需携带管理员 Token。

### 3.2 用例设计方法
- **正向用例：** 管理员 Token + 合法参数，验证返回 `code: 200` 且操作生效（新增、修改、删除）。
- **异常用例——资源校验：** 操作不存在的资源/分类 ID，验证返回 `code: 500` 并携带明确错误提示。

### 3.3 测试流程
1. 管理员登录获取 Token。
2. 先执行分类新增，为资源新增提供测试分类。
3. 执行资源 CRUD（新增 → 修改 → 修改不存在 → 删除）。
4. 执行分类 CRUD（新增 → 修改 → 修改不存在 → 删除）。

## 6. 通过/失败标准

**通过标准：**
- 管理员正向操作返回 `code: 200`，资源/分类新增、修改、删除生效。
- 操作不存在的资源或分类返回 `code: 500`，"资源不存在"/"分类不存在"。

**失败标准：**
- 正向操作未生效或返回非 200。
- 操作不存在资源/分类未拦截，返回误导性提示。
