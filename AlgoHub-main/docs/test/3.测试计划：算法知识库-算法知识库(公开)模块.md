## 1. 测试目标

- 验证算法知识库（公开）模块 6 个功能（分类树、子分类、搜索、详情、按分类浏览、按难度筛选）的正确性，确保公开用户可正常浏览和检索算法数据。
- 验证参数校验：非法 ID 类型、不存在的算法 ID 等异常输入被正确拦截。
- 验证搜索接口的空关键词拦截生效（Bug #6）。
- 验证边界表现：不存在分类返回空列表、超大 pageSize 无限制等优化空间。

## 3. 测试策略

### 3.1 测试类型
接口功能测试（黑盒），通过 Apifox 发送 HTTP 请求，校验 HTTP 状态码及响应 JSON 体。全部为公开接口，无需 Token。

### 3.2 用例设计方法
- **正向用例：** 合法参数，验证返回 `code: 200` 且数据结构正确（分类树、分页列表、算法详情）。
- **异常用例——参数校验：** 非法 ID 类型（abc）、不存在的算法 ID（99999），验证返回 `code: 500` 并携带错误提示。
- **异常用例——空关键词：** keyword 传入纯空格，验证返回 `code: 500`，"搜索关键字不能为空"（Bug #6）。
- **边界用例：** 不存在的分类 ID 返回空列表、不存在的子分类返回空数组、不存在的关键词返回空列表、超大 pageSize 无限制。

### 3.3 测试流程
1. 先执行分类树及子分类接口，确认分类数据结构正确。
2. 执行搜索接口正向/异常用例，重点验证纯空格拦截。
3. 执行详情、按分类浏览、按难度筛选，覆盖正常路径和边界值。
4. 不通过用例标注 ❌，同步追加至 Bug 清单。

## 6. 通过/失败标准

**通过标准：**
- 所有正向用例返回 `code: 200`，响应数据结构与接口文档一致。
- 非法 ID 类型、不存在算法 ID 返回 `code: 500` 并携带可读错误提示。
- 搜索纯空格关键词返回 `code: 500`，"搜索关键字不能为空"。

**失败标准：**
- 正向用例未返回 200 或数据缺失/错误。
- 纯空格搜索未被拦截，返回全表数据。
- 非法参数导致内部错误未给出明确提示。

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 搜索空关键词绕过 | 全表数据泄露、性能隐患 | 专项验证纯空格 keyword 拦截（TC-A13，Bug #6） |
| 超大 pageSize 无限制 | 内存溢出、数据库压力 | 记录为优化项（Bug 清单 优化#3），后续增加上限 |
| 不存在分类仅返回空列表 | 用户体验差，无法区分"分类不存在"与"无数据" | 记录为优化项（Bug 清单 优化#4），后续增加友好提示 |
