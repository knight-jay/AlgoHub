## 1. 测试目标

- 验证学习资源推荐（公开）模块 4 个功能（分类列表、搜索、按分类筛选、资源详情）的正确性，确保公开用户可正常浏览和检索学习资源。
- 验证参数校验：不存在的资源 ID 返回错误提示（Bug #7）。
- 验证搜索接口的空关键词拦截生效。

## 3. 测试策略

### 3.1 测试类型
接口功能测试（黑盒），通过 Apifox 发送 HTTP 请求，校验 HTTP 状态码及响应 JSON 体。全部为公开接口，无需 Token。

### 3.2 用例设计方法
- **正向用例：** 合法参数，验证返回 `code: 200` 且数据结构正确（分类列表、分页列表、资源详情）。
- **异常用例——空关键词：** keyword 传入纯空格，验证返回 `code: 500`，"搜索关键字不能为空"（Bug #7）。
- **异常用例——资源校验：** 不存在的资源 ID（99999），验证返回 `code: 500`，"资源不存在"。
- **边界用例：** 空分类返回空列表、不带搜索参数走默认分页。

### 3.3 测试流程
1. 先执行分类列表接口，确认分类数据结构正确。
2. 执行搜索接口正向/异常用例，重点验证纯空格拦截。
3. 执行按分类筛选、资源详情，覆盖正常路径和边界值。
4. 不通过用例标注 ❌，同步追加至 Bug 清单。

## 6. 通过/失败标准

**通过标准：**
- 所有正向用例返回 `code: 200`，响应数据结构与接口文档一致。
- 不存在资源返回 `code: 500`，"资源不存在"。
- 搜索纯空格关键词返回 `code: 500`，"搜索关键字不能为空"。

**失败标准：**
- 正向用例未返回 200 或数据缺失/错误。
- 纯空格搜索未被拦截，返回全量数据。
- 不存在资源未给出错误提示。
