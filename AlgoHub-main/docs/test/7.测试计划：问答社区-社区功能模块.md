## 1. 测试目标

- 验证问答社区模块 23 个子功能（共 22 个接口）的功能正确性，确保帖子发布/编辑/删除、评论/回复、点赞/收藏/关注、举报处理、管理员审核等核心流程符合设计预期。
- 验证接口的权限控制是否生效：未登录拦截（401）、学生/管理员角色隔离（403）、资源归属校验（非作者不可编辑/删除）。
- 验证边界条件和异常输入（空值、纯空格、不存在的资源 ID、越权操作）下的接口鲁棒性。
- 发现并记录功能缺陷，输出至 Bug 清单，跟踪修复状态。

## 3. 测试策略

### 3.1 测试类型
接口功能测试（黑盒），通过 Apifox 发送 HTTP 请求，校验 HTTP 状态码及响应 JSON 体。

### 3.2 用例设计方法
- **正向用例：** 合法输入 + 有效 Token，验证返回 `code: 200` 且业务数据完整。
- **异常用例——参数校验：** 空字符串、纯空格、缺失必填字段，验证返回 `code: 500` 并携带明确错误提示。
- **异常用例——鉴权校验：** 不带 Token，验证返回 `code: 401`。
- **异常用例——权限校验：** 学生访问管理员接口，验证返回 `code: 403`。
- **异常用例——资源校验：** 操作不存在的资源（id=99999），验证返回错误提示。
- **异常用例——归属校验：** 非作者编辑/删除他人帖子或评论，验证被拦截。

### 3.3 测试流程
1. 预先注册测试账号（学生 + 管理员），登录获取 Token。
2. 按模块顺序执行用例，记录实际结果并与预期对比。
3. 不通过用例标注 ❌，同步追加至 Bug 清单。
4. Bug 修复后执行回归测试，更新用例状态。

## 6. 通过/失败标准

**通过标准：**
- 所有正向用例返回 `code: 200`，响应数据结构与接口文档一致。
- 所有权限校验用例正确返回 401 或 403。
- 所有参数校验用例被正确拦截并返回可读的错误提示（`msg` 字段）。

**失败标准：**
- 正向用例未返回 200 或数据缺失/错误。
- 未登录/越权操作未被拦截（返回 200 但实际不应允许）。
- 异常输入未拦截导致内部错误或数据不一致。

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 测试数据被其他测试会话覆盖 | 用例结果不可靠 | 每次测试前确认数据状态，必要时重建测试数据 |
| Token 过期 | 鉴权用例误报为 401 | 测试开始前重新登录，确保 Token 有效期充足 |
| 越权校验漏测 | 安全漏洞遗留 | 每个涉及资源归属的接口，强制交叉使用非作者 Token 验证 |
| 搜索空关键词绕过 | 全表数据泄露 | 专门设计纯空格 keyword 用例（TC-C10）进行验证 |
